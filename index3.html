Frequently Asked Questions

The difference between var, let, and const?

`var` is function-scoped, hoisted, and can be reassigned. `let` and `const` are block-scoped and hoisted but not initialized. `let` allows reassignment, while `const` does not (though object/array properties can be modified). Use `let` for variables and `const` for constants. Avoid `var` in modern JavaScript.

The difference between map(), forEach(), and filter()?

`map()` creates a new array by applying a function to each element. `forEach()` executes a function for each element but does not return a new array. `filter()` creates a new array with elements that pass a condition. Use `map()` for transformation, `forEach()` for side effects, and `filter()` for filtering data.

Explain arrow functions and how they are different from regular functions?

Arrow functions are a concise syntax for writing functions in JavaScript. They do not have their own `this`, `arguments`, `super`, or `new.target`. Unlike regular functions, `this` in arrow functions is lexically bound (inherited from the surrounding scope). They cannot be used as constructors and lack a `prototype` property.

How JavaScript Promises work?

JavaScript Promises handle asynchronous operations. A Promise represents a value that may be available now, later, or never. It has three states: **pending**, **fulfilled**, or **rejected**. Use `.then()` for success, `.catch()` for errors, and `. finally()` for cleanup. Promises simplify chaining async operations and avoid callback hell. Example: `fetch().then().catch()`..

How closures work in JavaScript?

Closures in JavaScript allow a function to access variables from its outer (enclosing) scope even after the outer function has finished executing. They "remember" the environment in which they were created. Closures are useful for data encapsulation, creating private variables, and maintaining state in asynchronous operations. Example: `function outer() { let x = 1; return function inner() { console.log(x); }; }`..
